# VertexFX Admin Roles Overview

## Admin Accounts & Roles

---

## 1. Super Admin

- **Email**: `admin@vertexfx.com`
- **Password**: `VertexFx@2025`

**Full Role Description**:
The Super Admin is the highest-level account with **UNRESTRICTED ACCESS** to every feature and functionality within the VertexFX platform. This role can perform ANY duty that any other role (Compliance, Finance, Risk Manager, Support, etc.) can do, plus additional platform-wide management functions. This role is intended for the platform owner or senior management personnel only.

**Key Responsibilities & Permissions**:

- **ALL Compliance Officer Duties**:
  - KYC document review/approval/rejection
  - AML alert management
  - User account freezing/unfreezing
  - Compliance reporting
  - And all other compliance functions
- **ALL Finance Officer Duties**:
  - Deposit/withdrawal approval
  - Wallet balance adjustments
  - Financial reporting
  - Payment reconciliation
  - And all other financial functions
- **ALL Risk Manager Duties**:
  - Position limit configuration
  - Margin call management
  - High-risk position closure
  - Risk exposure reporting
- **ALL Support Staff Duties**:
  - User account support
  - Password resets
  - Ticket management
- **Plus Additional Super Admin Only Permissions**:
  - **Full User Management**: Create, read, update, delete (CRUD) any user account (including other admins, staff, and traders).
  - **Staff Account Creation**: Manage all staff roles (Compliance, Finance, Support, Risk Manager, other Super Admins).
  - **Platform Configuration**: Modify system settings, including:
    - Trading parameters (leverage limits, instrument configurations)
    - Fee structures
    - KYC/AML rules
    - Maintenance mode
  - **Audit & Logs**: View complete audit trail of all actions taken on the platform.
  - **API & Webhook Management**: Configure API keys, webhook endpoints, and third-party integrations.
  - **Backup & Restore**: Access to database backup and restoration functions.

**Typical Use Cases**:

- Onboarding new staff members
- Changing platform-wide policies
- Resolving critical issues that require elevated permissions
- Covering staff duties during absences (e.g., approving KYC if Compliance Officer is away)
- Auditing staff actions
- Handling emergency platform maintenance

## 2. Compliance Officer

- **Email**: `compliance@vertexfx.com`
- **Password**: `Compliance@2025`

**Full Role Description**:
The Compliance Officer is responsible for ensuring the platform adheres to regulatory requirements, verifying user identities, and monitoring for suspicious activity.

**Key Responsibilities & Permissions**:

- **KYC Document Review**: Review, approve, or reject user-submitted KYC (Know Your Customer) documents.
- **User Verification Status**: Update user verification levels (unverified → verified → fully verified).
- **AML Alert Management**:
  - Review AML (Anti-Money Laundering) alerts generated by the system
  - Assign alerts to staff members
  - Mark alerts as resolved or escalate
  - View user risk profiles
- **Transaction Monitoring**: Flag and review suspicious transactions for further investigation.
- **User Activity Logs**: Access detailed user activity histories.
- **Document Management**: Request additional documentation from users when needed.
- **Compliance Reporting**: Generate compliance reports for regulatory purposes.
- **Account Restrictions**:
  - Freeze/unfreeze user accounts (with audit trail)
  - Restrict trading privileges (cannot close positions without compliance approval)
- **Staff Notes**: Add compliance-related notes to user accounts (visible to other staff only).

**Typical Use Cases**:

- Daily KYC document review and approval
- Investigating unusual trading patterns
- Responding to regulatory inquiries
- Conducting periodic user risk assessments
- Ensuring all users meet minimum verification requirements

## 3. Finance Officer

- **Email**: `finance@vertexfx.com`
- **Password**: `Finance@2025`

**Full Role Description**:
The Finance Officer manages all financial operations on the platform, including deposits, withdrawals, wallet balances, and payment reconciliations.

**Key Responsibilities & Permissions**:

- **Deposit Review & Approval**:
  - Review manual deposits (bank transfers)
  - Approve/reject deposit requests
  - Match deposits to user accounts
- **Withdrawal Processing**:
  - Review and approve withdrawal requests
  - Process manual bank transfers
  - Verify user bank account details
  - Mark withdrawals as processed/failed/cancelled
- **Wallet Management**:
  - View all user wallet balances
  - Adjust balances (credit/debit) for error corrections
  - View transaction history for any user
- **Transaction Reconciliation**:
  - Match system transactions with payment gateway records
  - Resolve discrepancies in deposit/withdrawal amounts
- **Fee Management**:
  - Configure withdrawal and trading fees (per account tier)
  - View fee revenue reports
- **Payment Gateway Integration**:
  - Monitor payment gateway webhooks
  - Troubleshoot failed transactions
- **Financial Reporting**:
  - Generate daily/weekly/monthly financial reports
  - Track total deposits, withdrawals, and trading volume
  - Export financial data for accounting purposes

**Typical Use Cases**:

- Daily withdrawal approval queue processing
- Reconciling end-of-day payment gateway transactions
- Correcting accidental duplicate deposits
- Processing large withdrawal requests that require manual approval
- Generating monthly financial statements

## Additional Roles (Available in Platform, Not Seeded by Default)

### Risk Manager

- **Responsibilities**: Monitor trading risk, set position limits, manage margin calls, configure risk parameters for different account tiers.
- **Permissions**:
  - View all open positions and margin usage
  - Set instrument-specific position limits
  - Adjust margin requirements temporarily
  - Force-close high-risk positions
  - Generate risk exposure reports

### Support Staff

- **Responsibilities**: Handle user inquiries, provide technical support, assist with account issues.
- **Permissions**:
  - View user account details (without sensitive data like passwords)
  - Respond to support tickets
  - Reset user passwords (with user confirmation)
  - Unlock locked accounts
  - View basic transaction history
  - Cannot modify financial data or compliance settings

## Important Security Notes

1. Always change default passwords after first login
2. Enable 2FA for all staff accounts
3. Limit staff access to only what is necessary for their role (principle of least privilege)
4. Regularly review audit logs for suspicious staff activity
5. Never share admin credentials
